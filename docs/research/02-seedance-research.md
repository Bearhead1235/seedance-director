# 即梦 Seedance 2.0 深度调研报告

> 调研人：researcher-seedance | 完成时间：2026-02-24

## 一、平台能力全景

### 支持的生成模式

Seedance 2.0 支持四种模态输入：图像、视频、音频、文本，可自由组合。

| 模式 | 说明 |
|------|------|
| 纯文本生成 | 仅靠文字描述生成视频，无需上传任何素材 |
| 图生视频（首尾帧） | 上传首帧图+文字描述，简单入口 |
| 多模态混合（全能参考） | 图片+视频+音频+文本自由组合，核心入口 |
| 视频延长 | 对已有视频进行前向/后向延长 |
| 视频编辑 | 在已有视频上进行角色替换、剧情修改等 |

两大入口：
- 首尾帧入口：仅上传首帧图+提示词，简单快捷
- 全能参考入口：多模态组合输入，功能最强

## 二、技术参数与限制

### 核心参数表

| 维度 | 规格 |
|------|------|
| 图片输入 | jpeg/png/webp/bmp/tiff/gif，≤9张，单张<30MB |
| 视频输入 | mp4/mov，≤3个，总时长2-15秒，单个<50MB，分辨率480p-720p |
| 音频输入 | mp3/wav，≤3个，总时长≤15秒，单个<15MB |
| 文本输入 | 自然语言描述（中英文均可） |
| 混合上限 | 最多12个文件（图片+视频+音频合计） |
| 生成时长 | 4-15秒，可自由选择 |
| 分辨率 | 支持480p/720p/1080p/2K输出 |
| 宽高比 | 16:9（横屏）、9:16（竖屏）、1:1（方形） |
| 帧率 | 默认24fps |
| 声音输出 | 自带音效/配乐，支持口型同步 |

### 额度消耗

- 免费用户：每日60积分
- 月度会员：69元/月，1080积分
- 年度会员：659元/年（约55元/月）
- 生成消耗：15秒视频约90积分（约6积分/秒）

## 三、素材引用语法（@引用系统）

### 命名规范

| 类型 | 编号 | 范围 |
|------|------|------|
| 图片 | `@图片1` ~ `@图片9` | 最多9张 |
| 视频 | `@视频1` ~ `@视频3` | 最多3个 |
| 音频 | `@音频1` ~ `@音频3` | 最多3个 |

### 编号分配规则

1. 按上传顺序自动编号，同类型素材依次递增
2. 必须使用中文名称：`@图片1`（不是@img1）、`@视频1`（不是@video1）
3. 在输入框输入"@"会弹出素材选择列表
4. 鼠标悬停@引用可预览素材，避免引用错误

### 引用使用方式示例

```
@图片1 作为首帧
@图片2 作为尾帧
@图片3 作为角色形象参考
参考@视频1的运镜效果
参考@视频1的打斗动作
背景音乐参考@音频1
旁白的音色参考@视频1
```

### 关键原则

- 必须明确说明每个素材的用途，不要让模型猜
- 区分「参考」和「编辑」：参考是借鉴风格/动作生成新内容，编辑是在原素材基础上修改
- 素材多时反复检查@引用，避免把图片当视频引用

## 四、十大核心能力详解

### 1. 纯文本生成

提示词模式：`(主体描述) + (动作序列) + (环境/光影) + (镜头语言) + (风格关键词)`

示例：
```
镜头跟随黑衣男子快速逃亡，后面一群人在追，镜头转为侧面跟拍，人物惊慌撞倒路边的水果摊爬起来继续逃，人群慌乱的声音。
```

### 2. 一致性控制（角色/产品/场景统一）

提示词模式：`[角色]@图片N + [动作/剧情描述] + [场景]@图片N + [运镜/光影]`

示例（角色一致性）：
```
男人@图片1下班后疲惫的走在走廊，脚步变缓，最后停在家门口，脸部特写镜头，男人深呼吸，调整情绪，收起了负面情绪，变得轻松，然后特写翻找出钥匙，插入门锁，进入家里后，他的小女儿和一只宠物狗，欢快的跑过来迎接拥抱，室内非常的温馨，全程自然对话
```

### 3. 运镜与动作精准复刻

提示词模式：`参考@视频1的[运镜/动作/节奏] + [主体]@图片N + [场景描述]`

### 4. 创意模板/特效复刻

提示词模式：`参考@视频1的[特效/转场/创意] + 将[元素]替换为@图片N + [补充说明]`

### 5. 剧情创作/补全

提示词模式：`[分镜脚本/图片内容描述] + [演绎方式] + [音效/台词要求]`

### 6. 视频延长（前向/后向）

提示词模式：
- 后向：`将@视频1延长[X]s + [新增内容描述]`
- 前向：`向前延长[X]s + [前置剧情描述]`

重要：生成时长应选择"新增部分"的时长

### 7. 声音控制（音色克隆+台词）

提示词模式：`[画面描述] + 音色/旁白参考@视频1 + [台词内容用引号标注]`

### 8. 一镜到底

提示词模式：`一镜到底 + @图片1@图片2... + [连续场景描述] + 全程不要切镜头`

### 9. 视频编辑（换元素/改剧情）

提示词模式：
- 角色替换：`将@视频1中的[A]换成@图片1 + [其他修改]`
- 剧情颠覆：`颠覆@视频1的剧情 + [新剧情描述]`

### 10. 音乐卡点

提示词模式：`@图片1@图片2...@图片N + 参考@视频1的画面节奏/卡点 + [画面风格说明]`

## 五、提示词最佳实践

### 黄金公式

```
[主体] + [动作] + [场景] + [风格] + [情绪]
```

### 时间戳分镜法（15秒长视频强烈推荐）

```
0-3秒：[画面描述 + 镜头语言]
4-8秒：[画面描述 + 镜头语言]
9-12秒：[画面描述 + 镜头语言]
13-15秒：[画面描述 + 镜头语言]
```

### 技术参数前置法

```
2.35:1，24fps，15秒，8镜头硬切
霓虹高饱和冷暖对比，现代舞台
浅景深突出动作，声音设计优先
禁止文字logo水印
```

### 禁止项声明

```
禁止：任何文字、字幕、LOGO或水印
```

### 时长策略

| 时长 | 建议 |
|------|------|
| 4-8秒 | 产品展示、单个动作，不需要时间戳分镜 |
| 9-12秒 | 完整短场景，可选用时间戳分镜 |
| 13-15秒 | 完整叙事，强烈推荐时间戳分镜 |
| >15秒 | 分段生成+视频延长拼接 |

## 六、已知限制

### 写实人脸限制（最重要）

- 暂不支持上传含有写实真人清晰脸部的素材
- 绕过方法：用AI生成的写实人像替代，或走动漫/半写实风格

### 常见失败模式

| 失败模式 | 规避方法 |
|---------|---------|
| @引用标错 | 写完提示词后检查每个@引用 |
| 矛盾指令 | 不要让两段视频做矛盾的参考 |
| 提示词太模糊 | 使用具体画面描述 |
| 风格不一致 | 图片风格必须与视频主题契合 |
| 参考视频太高清 | 压缩到480p-720p范围 |
| 同一输入效果波动大 | 多生成几次挑选最佳 |

## 七、关键洞察

1. 控制力是核心价值：人物稳定、动作连贯、镜头合理、节奏可调
2. @引用系统是关键交互：标注清楚是出片质量的决定性因素
3. 时间戳分镜法是高级用户必备：对15秒长视频效果提升巨大
4. 写实人脸是最大限制：目前只能用AI生成人像或动漫风格绕过
5. 效果波动性较大：同样的输入多次生成可能效果差异很大
